# Code 节点硬约束分析

> 基于 Dify 前端 (web/) + 后端 (api/) + graphon 源码分析

## 1. 函数签名约束（**MUST**）

### 函数名必须为 `main`

| 语言 | 格式要求 | 正则检测（前端） |
|------|---------|----------------|
| Python3 | `def main(param1: type, ...) -> dict:` | `def\s+main\s*\((.*?)\)` |
| JavaScript | `function main(param1, ...) {}` | `function\s+main\s*\((.*?)\)` |

**源码位置**：
- 前端正则：`web/app/components/workflow/nodes/code/code-parser.ts` L7-14
- 后端默认模板：`api/core/helper/code_executor/python3/python3_code_provider.py` L11-20

### 参数名必须匹配 variables

函数参数名 **必须** 与 `variables` 数组中的 `variable` 字段一一对应：

```python
# variables 配置:
# [{"variable": "email", "value_selector": ["node1", "output"]}]
# 
# 函数签名必须:
def main(email: str) -> dict:
    return {"result": email}
```

前端 `code-parser.ts` 提取参数名逻辑：
```typescript
params.push(...match[1].split(',')
    .map(p => p.trim())
    .filter(Boolean)
    .map(p => p.split(':')[0].trim()))  // 取冒号前的参数名
```

## 2. 返回值约束（**MUST**）

- **必须返回字典/对象**（`return {...}`）
- 返回字典的 key **必须匹配 outputs 中定义的变量名**

```python
# outputs 配置:
# {"result": {"type": "string", "children": null}}
#
# 返回值必须:
def main(arg1: str) -> dict:
    return {"result": arg1}  # key "result" 对应 outputs 中的 "result"
```

## 3. variables 结构

```yaml
variables:
  - variable: arg1           # 参数名（必须与函数参数匹配）
    value_selector:          # 数据来源路径
      - node_id
      - output_key
    value_type: string       # 可选：类型标注
```

**后端定义** (`api/core/helper/code_executor/code_node_provider.py`)：
```python
class VariableConfig(TypedDict):
    variable: str
    value_selector: Sequence[str | int]
```

**默认值**：
```python
variables = [
    {"variable": "arg1", "value_selector": []},
    {"variable": "arg2", "value_selector": []},
]
```

## 4. outputs 结构

```yaml
outputs:
  result:                    # 输出变量名（必须与 return dict key 匹配）
    type: string             # 类型（见下表）
    children: null           # 预留嵌套支持（目前始终 null）
```

### 允许的输出类型

| 类型 | 说明 |
|------|------|
| `string` | 字符串 |
| `number` | 数字 |
| `boolean` | 布尔值 |
| `object` | 对象 |
| `array[string]` | 字符串数组 |
| `array[number]` | 数字数组 |
| `array[object]` | 对象数组 |
| `array[boolean]` | 布尔数组 |

**源码位置**：`graphon/nodes/code/entities.py` L19-28 (`_ALLOWED_OUTPUT_FROM_CODE`)

## 5. code_language 枚举

| 值 | 后端执行引擎 |
|----|-------------|
| `python3` | Python3 沙箱 |
| `javascript` | Node.js 沙箱 |
| `jinja2` | Python3 沙箱（Jinja2 模板） |

**注**：前端 UI 仅显示 `python3` 和 `javascript`。`json` 仅用于前端展示，不执行。

## 6. 执行限制（运行时）

| 限制 | 配置项 |
|------|--------|

| 数字范围 | `CODE_MAX_NUMBER` / `CODE_MIN_NUMBER` |

## 7. 默认代码模板

### Python3

```python
def main(arg1: str, arg2: str) -> dict:
    return {
        "result": arg1 + arg2,
    }
```

### 对应默认配置

```yaml
code_language: python3
code: |
  def main(arg1: str, arg2: str) -> dict:
      return {
          "result": arg1 + arg2,
      }
variables:
  - variable: arg1
    value_selector: []
  - variable: arg2
    value_selector: []
outputs:
  result:
    type: string
    children: null
```

## 8. 前端"同步函数签名"功能

前端 Code 节点面板提供"同步函数签名"按钮（`panel.tsx`），自动：
1. 解析代码中的 `def main(...)` 参数 → 生成 `variables` 配置
2. 解析 `return {...}` 的 key → 生成 `outputs` 配置

## 9. 本工具的处理方式

### 编辑器默认值 (`editor.py`)

创建 Code 节点时自动生成完整的默认代码、variables、outputs，对齐 Dify 官方 `get_default_config()。

### 校验规则 (`node_data_validator.py`)

| 规则 | 级别 | 说明 |
|------|------|------|
| `code_language` 必须为 python3/javascript | error | 枚举值检查 |
| 代码必须包含 `def main(...)` 或 `function main(...)` | error | 函数签名检查 |
| 函数参数名需匹配 variables | warning | 交叉校验 |
| outputs 必须是 dict 且非空 | warning | 结构检查 |
| output type 必须在允许列表内 | error | 类型约束 |
| variables 项必须有 variable 名 | error | 完整性检查 |

### 关键实现细节

Code 节点的 `variables` 和 `outputs` 格式与 Start/End 节点不同：
- Start 节点 variables: `list[StartVariable]`（含 label, type, required 等）
- Code 节点 variables: `list[dict]`（含 variable, value_selector）
- End 节点 outputs: `list[OutputVariable]`
- Code 节点 outputs: `dict[str, {type, children}]`

因此在 `editor.py` 中，Code 节点的 variables/outputs 存储在 `__pydantic_extra__` 中，
避免与 NodeData 的类型化字段冲突。
