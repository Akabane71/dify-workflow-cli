# Dify 五种应用类型对比研究

> 基于 dify-test 源码逆向分析，版本 ≥ 0.6.0

## 一、五种应用类型总览

Dify 平台提供 5 种面向用户的应用类型（`AppMode` 枚举，定义于 `api/models/model.py`）：

| # | UI 名称 | `app.mode` 值 | 本质架构 |
|---|---------|--------------|---------|
| 1 | **工作流** | `workflow` | Workflow-based |
| 2 | **Chatflow** | `advanced-chat` | Workflow-based |
| 3 | **聊天助手** | `chat` | ModelConfig-based |
| 4 | **Agent** | `agent-chat` | ModelConfig-based |
| 5 | **文本生成** | `completion` | ModelConfig-based |

另有 `channel`、`rag-pipeline` 两个内部模式，不面向用户。

---

## 二、两大底层架构

这 5 种类型本质上分为 **两种完全不同的架构**：

### A. Workflow 架构（工作流 / Chatflow）

- **配置实体**: `WorkflowUIBasedAppConfig`
- **存储方式**: `App.workflow_id` → `Workflow` 表（包含 graph、features、环境变量等）
- **DSL 结构**: 含 `workflow:` 字段（graph + features + environment_variables + conversation_variables）
- **不含** `model_config:` 字段
- **运行器**: `WorkflowAppGenerator` / `AdvancedChatAppGenerator`

### B. ModelConfig 架构（聊天助手 / Agent / 文本生成）

- **配置实体**: `EasyUIBasedAppConfig`
- **存储方式**: `App.app_model_config_id` → `AppModelConfig` 表（包含 model、prompt、agent_mode 等）
- **DSL 结构**: 含 `model_config:` 字段（model + pre_prompt + agent_mode + dataset_configs 等）
- **不含** `workflow:` 字段
- **运行器**: `ChatAppGenerator` / `AgentChatAppGenerator` / `CompletionAppGenerator`

**两个 FK 字段互斥**：`app_model_config_id` 和 `workflow_id` 在 `App` 模型中都是 `nullable=True`，由 `mode` 决定使用哪一个。

## 三、各类型详细对比

### 3.1 工作流 (`workflow`)

**定位**: 单轮任务编排，无对话能力。

- 用户通过可视化画布拖拽节点（Start → 处理节点 → End）
- 起始节点: `START`（或触发器: webhook / schedule / plugin）
- 终止节点: `END`（定义输出变量）
- WorkflowType: `workflow`
- **无对话上下文** — 不支持 `conversation_id`、无 memory
- 适用场景: 批量处理、定时任务、API 驱动的数据管道

### 3.2 Chatflow (`advanced-chat`)

**定位**: 带对话能力的工作流，兼具画布编排 + 多轮对话。

- 与工作流共用同一套 graph 引擎
- 起始节点: `START`（含 `sys.query`、`sys.conversation_id`、`sys.dialogue_count`）
- 终止节点: **`ANSWER`**（不是 `END`）— 输出到聊天界面
- WorkflowType: `chat`（注意！不是 `workflow`）
- **支持多轮对话** — 创建 `Conversation` + `Message` 记录
- **支持 conversation_variables** — 跨轮次持久化变量
- 适用场景: 复杂聊天机器人、带知识库检索的对话、多步骤客服流程

### 3.3 聊天助手 (`chat`)

**定位**: 简单聊天机器人，无需画布。

- 直接配置模型 + 系统 Prompt + 知识库
- 通过 `AppModelConfig` 存储: `model`、`pre_prompt`、`user_input_form`、`dataset_configs`
- **支持多轮对话**
- 无 workflow graph，无 environment_variables
- 如果开启 `agent_mode.enabled=true` 且策略为 `function_call`/`react`，会自动升级为 Agent 模式
- 适用场景: 快速搭建简单对话机器人

### 3.4 Agent (`agent-chat`)

**定位**: 具备工具调用能力的智能体。

- 数据模型与 Chat 基本一致，额外含 `agent_mode` 配置
- `agent_mode: { enabled: true, strategy: "function_call"|"react", tools: [...] }`
- **仅支持流式输出** — 不支持 blocking 模式
- 可调用外部工具进行多步推理
- 适用场景: 需要调用 API、数据库、计算器等外部工具的智能助手

### 3.5 文本生成 (`completion`)

**定位**: 单轮文本生成，无对话。

- 用户提交表单 → 模型生成 → 返回结果
- **无对话上下文** — 不支持 `conversation_id`
- 独有功能: **More-like-this**（生成相似变体）
- 默认模板含 `query` 段落变量 + `pre_prompt: "{{query}}"`
- 适用场景: 文案生成、翻译、摘要等一次性文本任务
