# Dify 源码架构：应用类型的实现分层

> 从源码角度理解 5 种应用类型的技术实现

## 一、枚举定义层

### AppMode（`api/models/model.py`）

```python
class AppMode(StrEnum):
    COMPLETION = "completion"       # 文本生成
    WORKFLOW = "workflow"           # 工作流
    CHAT = "chat"                   # 聊天助手
    ADVANCED_CHAT = "advanced-chat" # Chatflow
    AGENT_CHAT = "agent-chat"       # Agent
    CHANNEL = "channel"             # 内部
    RAG_PIPELINE = "rag-pipeline"   # 内部
```

### WorkflowType（`api/models/workflow.py`）

```python
class WorkflowType(StrEnum):
    WORKFLOW = "workflow"
    CHAT = "chat"

    @classmethod
    def from_app_mode(cls, app_mode):
        return cls.WORKFLOW if app_mode == AppMode.WORKFLOW else cls.CHAT
```

**映射关系**: 5 种 AppMode → 2 种 WorkflowType（所有非 WORKFLOW 模式统一为 CHAT）

### 前端枚举（`web/types/app.ts`）

```typescript
export enum AppModeEnum {
  COMPLETION = 'completion',
  WORKFLOW = 'workflow',
  CHAT = 'chat',
  ADVANCED_CHAT = 'advanced-chat',
  AGENT_CHAT = 'agent-chat',
}
```

## 二、数据模型层

### App 模型（`api/models/model.py`）

```
App
├── mode: AppMode                    # 应用类型
├── app_model_config_id: UUID?       # → chat/agent/completion 使用
├── workflow_id: UUID?               # → workflow/advanced-chat 使用
└── ...
```

**两个 FK 互斥**: 由 `mode` 决定使用哪一个。

### 存储分支

```
App(mode=workflow)       → Workflow 表    → graph JSON + features JSON
App(mode=advanced-chat)  → Workflow 表    → graph JSON + features JSON
App(mode=chat)           → AppModelConfig → model + pre_prompt + dataset_configs
App(mode=agent-chat)     → AppModelConfig → model + pre_prompt + agent_mode
App(mode=completion)     → AppModelConfig → model + pre_prompt + user_input_form
```

## 三、配置实体层（`api/core/app/app_config/entities.py`）

```
BaseAppConfig
├── EasyUIBasedAppConfig          ← chat / completion
│   ├── model: ModelConfigEntity
│   ├── prompt_template: PromptTemplateEntity
│   ├── dataset: DatasetEntity
│   └── AgentChatAppConfig        ← agent-chat 扩展
│       └── agent: AgentEntity
│
└── WorkflowUIBasedAppConfig      ← workflow / advanced-chat
    └── workflow_id: str
```

## 四、运行器层（`api/core/app/apps/`）

```
apps/
├── workflow/           → WorkflowAppGenerator
│                         + WorkflowAppRunner
├── advanced_chat/      → AdvancedChatAppGenerator
│                         + AdvancedChatAppRunner
├── chat/               → ChatAppGenerator
│                         + ChatAppRunner
├── agent_chat/         → AgentChatAppGenerator
│                         + AgentChatAppRunner
├── completion/         → CompletionAppGenerator
│                         + CompletionAppRunner
└── common/             → 共享工具
```

**继承关系**:
```
BaseAppGenerator
├── MessageBasedAppGenerator      ← 有对话能力的生成器
│   ├── ChatAppGenerator
│   ├── AgentChatAppGenerator
│   ├── CompletionAppGenerator
│   └── AdvancedChatAppGenerator
└── WorkflowAppGenerator          ← 纯工作流，无对话
```

```python
if app_mode in {AppMode.ADVANCED_CHAT, AppMode.WORKFLOW}:
```

## 六、选型决策树

```
需要可视化节点编排吗？
├── 是 → 需要多轮对话吗？
│        ├── 是 → Chatflow (advanced-chat)
│        └── 否 → 工作流 (workflow)
└── 否 → 需要工具调用吗？
         ├── 是 → Agent (agent-chat)
         └── 否 → 需要多轮对话吗？
                  ├── 是 → 聊天助手 (chat)
                  └── 否 → 文本生成 (completion)
```
