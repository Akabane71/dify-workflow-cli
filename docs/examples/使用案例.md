# 案例2:复杂的 企业客服与工单处理工作流

这个流程里包含：

* 开始节点
* 输入预处理
* 意图识别
* 条件分支
* 知识库检索
* 工具调用【http】
* LLM 回复生成
* 低置信度转人工 【http】
* 循环补问
* 工单创建
* 通知
* 结束

### mermaid图
**大致的流程图:**
```mermaid
flowchart TD
    A([Start]) --> B[输入清洗 / 敏感检测]
    B --> C{意图识别}

    %% 咨询类
    C -->|咨询类| D[知识库检索]
    D --> E{检索结果置信度 >= 阈值?}
    E -->|是| F[LLM生成正式回复]
    F --> Z1([End])

    E -->|否| G[补问节点 / 澄清问题]
    G --> H{用户是否补充信息?}
    H -->|是| D
    H -->|否| I[转人工]
    I --> Z2([End])

    %% 订单类
    C -->|订单类| J[调用订单系统 API]
    J --> K{订单诉求类型}
    K -->|退款| L[执行退款规则校验]
    K -->|催单| M[查询物流与履约状态]
    K -->|改址| N[校验是否允许修改地址]
    L --> O[LLM生成订单答复]
    M --> O
    N --> O
    O --> Z3([End])

    %% 故障类
    C -->|故障类| P[提取报错码 / 日志]
    P --> Q[调用监控 / 服务状态工具]
    Q --> R{是否命中已知故障?}
    R -->|是| S[输出修复方案与文档链接]
    S --> Z4([End])
    R -->|否| T[创建技术工单]
    T --> U[通知责任人 / 群机器人]
    U --> Z5([End])

    %% 投诉类
    C -->|投诉类| V[情绪分析 / 风险评级]
    V --> W{是否高风险?}
    W -->|是| X[直接转人工主管]
    X --> Z6([End])
    W -->|否| Y[LLM安抚并收集关键信息]
    Y --> AA[创建投诉工单]
    AA --> Z7([End])
```
